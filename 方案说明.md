# 东部智脑-运营管理综合调度平台 Web 原型方案

## 1. 首页驾驶舱定位

首页作为大区运营管理总控驾驶舱，当前阶段只承担两件事：

- 看功能：以八大调度模块作为首页主体，快速识别平台能力边界。
- 看指标：每个模块只展示当前最关键的 4 个指标，点击后进入单项业务页面查看明细。

跨专题风险、调度动作、任务派发等内容不放在首页展开，统一沉淀到对应专题页，尤其是“运营风险调度”页面。

## 2. 八大调度模块

### 数字化星级调度

核心指标：

- 大区当前整体星星数
- 大区全年星星数目标
- 全年整改任务总数
- 当前已完成任务数量
- 当前进行中任务数量
- 当前逾期未开始任务数量

页面重点：

- 星级目标差距
- 整改任务闭环
- 重复问题和基础问题治理
- 专项提升行动

### 集约化调度

核心指标：

- 大区整体进度
- 当前已完成任务数量
- 当前进行中任务数量
- 当前逾期未开始任务数量
- 共享岗位覆盖率
- 集约化收益测算

页面重点：

- 集约化任务推进
- 区域进度差异
- 共享能力建设
- 人效和收益复盘

### 工艺AI调度

核心指标：

- 大区整体节点进度
- 滞后节点数量
- AI优化建议数
- 建议采纳率
- 异常参数识别次数
- 工艺稳定指数

页面重点：

- 工艺节点进度
- 滞后节点穿透
- AI建议执行闭环
- 水质、水量、药耗、能耗联动分析

### 专业化公司调度

核心指标：

- 专业公司覆盖数
- 本月支撑任务
- 响应达成率
- 跨区域协同事项
- 服务评价均分
- 待协调资源

页面重点：

- 专业化公司能力池
- 支撑任务派发与闭环
- 跨区域资源协同
- 服务评价与成果沉淀

### 工单管理调度

核心指标：

- 本月工单总数
- 工单闭环率
- 超时工单数量
- 平均响应时长
- 重复工单占比
- 一次解决率

页面重点：

- 工单队列
- 超时督办
- 重复工单根因分析
- 闭环质量抽检

### 经营管理调度

核心指标：

- 经营目标进度
- 预算执行金额
- 节约金额
- 超预算事项
- 回款完成率
- 经营风险事项

页面重点：

- 经营指标总览
- 预算与回款偏差预警
- 超预算事项闭环
- 经营风险穿透分析

### 安全管理调度

核心指标：

- 隐患整改率
- 未闭环隐患数量
- 高风险作业数量
- 培训覆盖率
- 应急演练完成率
- 安全事件数量

页面重点：

- 隐患整改闭环
- 高风险作业审批和提醒
- 培训演练完成情况
- 安全台账检查

### 运营风险调度

核心指标：

- 当前预警事项
- 高风险事项数量
- 处理中风险数量
- 本月风险关闭率
- 平均关闭时长
- 重复风险事项数量

页面重点：

- 跨专题风险池
- 风险分级和派发
- 重复风险根因分析
- 风险关闭质量复盘

## 3. 页面结构建议

每个专题页统一采用以下结构：

- 顶部：专题名称、业务说明、返回驾驶舱
- 指标区：4 到 6 个核心指标
- 方案区：目标牵引、过程管控、闭环复盘三类说明
- 台账区：任务事项、责任部门、状态、完成率、风险等级
- 右侧判断区：调度判断、趋势图、风险与动作

## 4. 后续接数据建议

后续可以把当前前端样例数据替换为接口数据，建议优先定义四类接口：

- `GET /dashboard/summary`：首页综合指标
- `GET /modules`：八大专题基础配置和卡片指标
- `GET /modules/{id}/metrics`：专题页指标、趋势和任务台账
- `GET /risks`：跨专题风险池

第一阶段先保证“看得清、点得进、能解释”；第二阶段再补规则引擎、权限体系、真实数据接口和移动端适配。
